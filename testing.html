Test Plan

Unit Testing

First, we test our prediction models to ensure sufficient accuracy using the mean absolute scaled error (MASE) method. Then, we test the application itself.
  • Login page
    • Loads correctly when application is opened
    • Accepts only valid usernames and password combinations
  • Home Screen
    • Loads correctly after user logs in
    • Each button/link on the home screen redirects the user to the correct page
    • Log out button should correctly log the user out
  • Model Screens
    • Load correctly from home screen
    • Each input field accepts only valid information (ex: temperature field should not accept non-numeric characters)
    • Submit button updates display graph on screen
    • Display graph has appropriate ranges on its axes which are properly labeled
    • Graph’s prediction display is in accordance with numeric predictions (eg: if model predicts 80 customers in a given time frame, the graph should not have a marker at 40 or a line going through 100)
    • Tabs at the top of the screen should redirect user to the correct page
    • Return button should redirect user to the home page

Integration/Function Testing

  • The user can login using valid credentials, and is redirected to the home screen
  • The user is given a warning message and remains at the login page when invalid credentials are given
  • All navigation components (buttons, icons, tabs) redirect the user to the correct location
  • Predictions are generated in the model screens when the user enters valid input parameters and presses submit
  • Informative error messages are generated when the user supplies invalid input in the model screens
  • Predictions given by models in the application match expected predictions given the entered parameters

System Testing

Because the product is a web-based application, compatibility issues primarily arise with respect to the user’s web browser, as opposed to the computer hardware or operating system being used.
  • Ensure that the application runs in the most recent versions of Mozilla Firefox and Google Chrome
  • Verify basic compatibility for other web browsers, including Internet Explorer 11, Microsoft Edge, and Safari
  • Verify the application loads in Microsoft Windows and OS X

User Testing

The marketed users of this product are cafe/store owners or managers, who may have limited technical experience. The objective of user testing is then to ensure that the product is informative and intuitive enough to be used effectively by adults with non-technical backgrounds.
  • The user feels comfortable navigating between screens within the web application
  • The user is content with the aesthetic of the application
  • The user can understand what input is necessary for a model’s prediction, and how to provide valid input
  • The user can interpret a model’s output easily without knowledge of the particular model
  • Error messages are informative and helpful in correcting the user
